## 一、Skill 脚本自动驱动（一行命令，完全自动化）

### ✅ 选项 A：搬开发工具缓存到 D 盘（强烈推荐）

```powershell
cd C:\.trae\skills\c-drive-cleaner
.\migrators\migrate-dev-caches.ps1 -TargetDrive D:
```

**脚本会自动做这些**：

| 工具 | 搬什么 | 安全吗？ | 效果 |
|------|--------|---------|------|
| **npm** | `npm cache`（下载包的缓存） | ✅ 纯缓存，配置文件在项目里 | 以后 `npm install` 缓存走 D 盘 |
| **pip** | `pip cache`（下载包的缓存） | ✅ 纯缓存 | 以后 `pip install` 缓存走 D 盘 |
| **yarn** | `yarn cache`（下载包的缓存） | ✅ 纯缓存 | 以后 `yarn add` 缓存走 D 盘 |
| **Gradle** | `GRADLE_USER_HOME`（依赖缓存） | ✅ 纯缓存 | 以后 `gradle build` 缓存走 D 盘 |
| **Go** | `GOMODCACHE`（模块缓存） | ✅ 纯缓存 | 以后 `go build` 缓存走 D 盘 |

**执行后效果**：
```
D:\dev-cache\
├── npm\       ← 以后 npm install 缓存下到这
├── pip\       ← 以后 pip install 缓存下到这
├── yarn\      ← 以后 yarn add 缓存下到这
├── gradle\    ← 以后 gradle build 依赖下到这
└── ...
```

**⚠️ 注意**：运行完后**重启终端**让环境变量生效。你现有的缓存已经被清理过了，所以迁移后 C 盘不会再多出空间，但**以后永远不增长**。

## 二、不推荐迁移的项目（迁移复杂 > 收益）

### ❌ 搬微信/飞书缓存 — 🔴 复杂，不建议

| 应用 | 路径 | 典型大小 | 为什么不建议搬 |
|------|------|---------|--------------|
| **微信** | `%APPDATA%\Tencent\WeChat` | ~4.8 GB | ❌ 目录结构极其复杂，**缓存、聊天记录、账号配置完全混在一起**，没有清晰的缓存目录边界。一旦搬错可能导致聊天记录丢失、登录态失效。目前没有自动化迁移脚本。 |
| **飞书** | `%APPDATA%\LarkShell` | ~4.8 GB | ❌ 同上，已清理过一次，**不建议再搬**。 |

> 💡 **更好的做法**：直接用 `.\cleaners\clean-apps.ps1 -RiskLevel safe` 定期清理它们的缓存，简单安全。

### ❌ 其它不推荐迁移的项目

| 项目 | 为什么不推荐搬 | 更好的做法 |
|------|--------------|-----------|
| **Trae / Trae CN** | 已清理 15 GB 缓存，剩下的主要是登录态和配置 | ✅ **已清理完成，不动** |
| **JetBrains** | 快捷键/主题/插件/试用信息混在一起 | ✅ **定期用 clean-safe 清缓存即可** |
| **Windows 系统文件** | ❌ **绝对不要碰** | Windows 自带清理就可以 |

## 三、你该怎么选？

```
你现在的 C 盘 38 GB 剩余，已经比较健康了。
你想：
├── 👉 A：跑一行脚本，以后开发缓存不再占 C 盘（强烈推荐）
│   → .\migrators\migrate-dev-caches.ps1 -TargetDrive D:
│   → Skill 自动完成，纯安全，一劳永逸
│
└── 👉 B：什么都不做，每两周跑一次 clean-safe 维护
    → 当前 74.4% 使用率，38 GB 剩余，已经很健康了
```

### 我的建议

**推荐你选 A**，原因：
1. 一行命令，完全自动化
2. 安全无风险（只改环境变量，不挪文件）
3. 一劳永逸——以后 C 盘不再被开发缓存占满
4. 你当前的 38 GB 剩余已经够用了

**不推荐你搬 JetBrains 和 TRAE**，因为它们配置和应用数据混在一起，丢了得不偿失。目前的 38 GB 剩余 + 每两周跑一次 clean-safe，C 盘可以长期维持在 70-75% 的健康状态。

## 五、如果选 A，执行完要验证

```powershell
# 验证 npm 缓存是否已指向 D 盘
npm config get cache
# 应该输出: D:\dev-cache\npm

# 验证 pip 缓存
echo %PIP_CACHE_DIR%
# 应该输出: D:\dev-cache\pip
```
