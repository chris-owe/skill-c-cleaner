# CleanSight 测试方法论

> **定义各模式的测试方法和验收标准。**

---

## 一、测试框架

### 测试范围

| 测试层 | 覆盖内容 | 测试方式 |
|--------|---------|---------|
| 单元测试 | 单个 .ps1 脚本的功能正确性 | 手动执行 + 结果验证 |
| 集成测试 | 多脚本组合（analyze.ps1 主流程） | 完整运行 + 报告验证 |
| 场景测试 | 模拟真实用户使用流程 | 按场景执行 + 体验评估 |
| 性能测试 | 扫描耗时、Token 消耗 | 计时 + 数据记录 |

### 验收标准

| 维度 | 通过标准 |
|------|---------|
| 功能完整性 | 所有扫描类别正常输出结果 |
| 数据准确性 | 两次相同配置扫描结果偏差 <5% |
| 安全性 | 只读扫描不修改任何文件 |
| 性能 | 扫描耗时在预期范围内 |
| 输出格式 | console/markdown/json 三种格式正常 |

---

## 二、标准模式测试方法

### 配置

```powershell
.\analyze.ps1 -Categories "A,B,C,D,E,F,G,H,I,J,K,L" -OutputFormat markdown
```

### 验收清单

- [ ] 12 类别各自输出独立结果
- [ ] 每项显示：名称、大小、风险等级、建议
- [ ] AI 决策建议部分包含 7 项能力对比矩阵
- [ ] Tier 1 / Tier 2 分级合理
- [ ] 健康评分与实际使用率匹配
- [ ] 报告包含使用指南和可执行命令
- [ ] 报告文件名格式正确（`CleanSight-CS-{date}-{time}-{score}.md`）

---

## 三、快速模式测试方法

### 配置

```powershell
.\analyze.ps1 -Categories "A,B,C,D,F" -OutputFormat markdown
```

### 验收清单

- [ ] 6 类别输出正常
- [ ] 排除的类别（G-L）不出现
- [ ] 扫描耗时明显少于标准模式

---

## 四、极速模式测试方法

### 配置

```powershell
.\analyze.ps1 -Categories "B,F,G" -OutputFormat markdown
```

### 验收清单

- [ ] 3 类别输出正常
- [ ] 优先识别最大的占用项
- [ ] 定位准确率 > 90%（与标准模式对比）

---

## 五、自定义模式测试方法

### 配置

```powershell
.\analyze.ps1 -Categories "C,G" -OutputFormat markdown
```

### 验收清单

- [ ] 仅输出选定的类别
- [ ] 聚焦精度高（没有无关数据）
