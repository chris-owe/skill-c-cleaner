## 1、Skill 简介

**CleanSight** 是一个 AI 驱动的 C 盘智能决策顾问——**它不是清理工具，而是你的磁盘健康私人 AI 管家**。

通过深度分析 + 个性化建议 + 安全执行指导，CleanSight 帮助用户在理解"为什么"的基础上做出明智的磁盘空间管理决策。

### 核心能力矩阵

| 能力维度 | CleanSight | CCleaner | Windows磁盘清理 |
|---------|-----------|----------|----------------|
| **上下文感知** | ✅ 基于你的使用习惯动态分析 | ❌ 固定规则一刀切 | ❌ 固定规则 |
| **动态风险评估** | ✅ 告诉你为什么 + 替代方案 + 迁移指导 | ⚠️ 只说能删/不能删 | ❌ 无风险评估 |
| **知识赋能** | ✅ 教你根本解决方法一劳永逸 | ❌ 删完就结束 | ❌ 删完就结束 |
| **个性化学习** | ✅ 越用越懂你（v6.2） | ❌ | ❌ |
| **成本透明** | ✅ Token消耗实时显示（~¥0.02-0.07/次）| 免费 | 免费 |
| **安全机制** | ✅ 只读默认+WhatIf+回滚指南 | ⚠️ 有风险 | ⚠️ 有风险 |

### 三大核心优势

```
优势 1: 上下文感知
   传统: "临时文件可以删除"
   AI:   "检测到你是开发者，npm缓存6.8GB可以安全删除，
         但建议先确认本周没有Node.js项目要编译"
```

## 2、使用场景

### 为什么做 CleanSight？

作为开发者，我发现每次 C 盘空间不足时都要经历一套重复流程：
- 打开 CCleaner → 扫描5分钟 → 看到一堆"可以删除"的项 → 不敢乱点 → 手动百度每个项是什么 → 浪费1小时
- 或者用 Windows 磁盘清理 → 只有基础项 → 清完发现只释放了500MB → 问题没解决
- 最痛苦的是 Docker、npm、WSL 这些开发工具的缓存，传统工具要么不敢删，要么删错了导致环境损坏

### 之前遇到的麻烦

**麻烦 1: 信息不对称**
- 传统工具告诉我"可以删除 2.3GB 临时文件"，但不解释这2.3GB是什么、删了会不会影响我的工作
- 我不知道哪些是安全的、哪些有风险、哪些应该迁移而不是删除

**麻烦 2: 一刀切的扫描策略**
- 不管我是256GB小硬盘紧急状态（只剩8GB），还是1TB大硬盘日常维护（用了60%），都是同样的全量扫描
- 小磁盘等不起5分钟，大磁盘不需要那么详细的信息

**麻烦 3: 开发者特有的痛点**
- Docker 镜像占了45GB，但我不敢随便 `docker system prune` 因为怕删掉正在用的镜像
- npm 缓存清了下次 `npm install` 又会重新下载，而且可能影响项目编译
- WSL2 的.vhdx 文件只增不减，Windows 自带工具根本不识别

**麻烦 4: 每次都从零开始**
- 上周刚问过"Docker怎么清理"，今天又要重新问一遍
- AI 不记得我的偏好、我的习惯、我关注什么、我不关心什么
- 每次交互都是冷启动，无法积累认知

### 做出来之后省掉了什么

✅ **省掉信息不对称**: AI 会解释每项是什么、为什么可以删、有什么替代方案  
✅ **省掉无效等待**: 15秒快速预检→智能推荐模式→按需扫描（而不是盲目全扫）  
✅ **省掉误操作风险**: 动态风险评估+安全红线+回滚指南  
✅ **省掉重复学习成本**: v6.2的个性化系统记住你的习惯，越用越好用  
✅ **省掉 Token 成本**: 从4.5K降到3.2K（29%优化），因为越来越精准

## 3、创作过程

### 第一版 v6.0: 基础扫描引擎

最初只想做一个"比CCleaner更智能的磁盘分析工具"：

- 实现了 **12 类别只读扫描器**（scanners/ 目录）
  - A-系统隐藏 (hiberfil/pagefile/还原点)
  - B-临时缓存 (Temp/回收站/Update缓存)
  - C-开发缓存 (npm/pip/cargo/maven)
  - D-浏览器 (Chrome/Edge/Firefox)
  - E-F-L-G... 共覆盖100+应用签名
  
- 设计了 **风险等级体系**
  - ✅ 安全（纯临时/缓存）
  - ⚠️ 需确认（低风险）
  - ❌ 绝不触碰（系统核心）
  - 🔴 可疑未知（引导判断）

- 构建了 **完整的安全机制**
  - 只读默认（扫描脚本绝不写入）
  - WhatIf 模式（清理前预览）
  - 快照备份（safety/snapshot-before-after.ps1）
  - 回滚指南（safety/rollback-guide.md）

**踩过的坑**:
- 一开始做成"一键清理"工具，结果用户反馈"不敢点"
- 改成"只读分析+建议"后，用户反而更愿意采纳建议

### 第二版 v6.1: 智能扫描策略系统

发现不同用户的差异巨大，开始解决**效率、成本、体验的平衡问题**：

#### 新增快速预检（15-30秒）

```powershell
# 新增文件: scanners/scan-quick-diagnosis.ps1
.\scanners\scan-quick-diagnosis.ps1 -OutputFormat json
```

采集轻量级数据：
- C盘总容量、已用空间、使用率
- Windows Temp + 回收站大小
- TOP 5 用户文件夹
- Docker/WSL 是否存在及大致大小
- 开发工具快速检测
- **用户类型自动识别**（开发者/普通用户/DevOps）

#### 设计 5 种扫描模式

| 模式 | 适用场景 | 扫描类别 | 耗时 | Token成本(DeepSeek Flash) |
|------|---------|---------|------|--------------------------|
| 🚀 极速 | 时间紧/紧急救援 | B+F+G (3类) | **15-30秒** | **~2.8K (¥0.02)** |
| ⚡ 快速 | 日常周维护 | A+B+C+D+F (6类) | **1-2分钟** | **~4.5K (¥0.03)** |
| 🔬 标准 | 月度全面体检 | 除H外11类 | **3-5分钟** | **~10.2K (¥0.07)** |
| 🎯 自定义 | 明确目标 | 用户选择 | **可变** | **~1-8K** |
| 🧠 智能 | 不确定选什么 | AI动态决策 | **最优** | **~2-10K** |

#### 实现 3 阶段交互流程

```
Phase 1: 快速预检 (15-30秒, ~500 tokens)
   ↓ 输出结构化 JSON + 用户友好快照
   
Phase 2: 策略推荐 (~800-1200 tokens)
   ↓ AI基于预检结果给出个性化建议（"推荐⚡快速模式，因为..."）
   
Phase 3: 执行选定策略
   ↓ 显示进度 → 生成报告 → 支持追问
```

**关键突破**:
- 从"用户选择模式"进化到"AI推荐模式+用户确认"
- 加入Token成本透明化（每次交互都告知消耗）
- 引入预算预警机制（>10K/20K/30K 时分级提醒）

将 `c-drive-cleaner` skill 目录放到项目的 `skills/` 下，或在 Trae 中导入 ZIP 包。

### 触发方式

当用户询问以下问题时自动触发此技能：
- "C盘空间不足"、"C盘快满了"


## 7、总结与思考

### 最满意的地方

#### 1️⃣ 差异化定位："决策层"而非"执行层"

```
传统工具 = 执行层：帮你删文件（但不懂你）
CleanSight = 决策层：理解你 → 分析数据 → 智能建议 → 教你安全执行
```

这不是一个"更好的CCleaner"，而是一个**全新的产品形态**：
- 传统工具回答"能不能删"
- CleanSight 回答"该不该删、为什么、有没有更好的方法、如何从根本上解决"

#### 2️⃣ 智能扫描策略（v6.1 的突破）

解决了不同用户的巨大差异：
- 256GB 小磁盘 vs 1TB 大磁盘 → 不同扫描深度
- 紧急状态（<10GB）vs 日常维护（70%使用率）→ 不同响应速度
- 新手用户 vs DevOps 工程师 → 不同信息密度
- 时间紧迫（30秒）vs 时间充裕（30分钟）→ 不同模式选择
